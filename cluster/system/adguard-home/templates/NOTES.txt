Post-deploy checks:
  dig @{{ .Values.metallb.ipv4 }} google.com
  dig @{{ .Values.metallb.ipv4 }} diskstation.fritz.box

Router:
  Set Fritzbox DHCP DNS to {{ .Values.metallb.ipv4 }} and {{ .Values.metallb.ipv6 }}.

Storage (like official Docker -v flags):
  /opt/adguardhome/conf  -> PVC adguard-conf  (AdGuardHome.yaml, TLS certs)
  /opt/adguardhome/work  -> PVC adguard-work  (querylog, stats, filter data)

Admin login:
  Username: adguard.admin.username in values.yaml
  Password: the plaintext used with htpasswd

Seal password:
  htpasswd -bnBC 10 "" 'mypassword' | cut -d: -f2 | \
    kubectl create secret generic adguard-admin \
      --namespace adguard-home \
      --from-literal=passwordHash="$(htpasswd -bnBC 10 "" 'mypassword' | cut -d: -f2)" \
      --dry-run=client -o yaml | kubeseal -o yaml

  Full htpasswd lines (user:hash) also work — init strips the username prefix.

After password change, restart the pod:
  kubectl rollout restart deployment/adguard-home -n adguard-home
